Synopsis:
Support the ability for a broker to run "in process" with any client.

Quick Questions:
  • Difficulity: Easy

Description:
A new service will be added to the broker that allows messages from the client to be directly routed inside the client.

A client application will start the broker "in process" using supported api's and then send messages to this service using a different connection implementation. `

Requestor:
JMSJCA RA and OpenMQ users

Non-feature:
Does not support memory management or flow control for the broker

Dependencies:
None

Phases:
Single phase

Direction of Implementation:
Code will be development in-house. Functionality will be split between broker and client engineers.

Interfaces:
Will add new client apis

Test Impact:
Will be easy to test. All tests can be automated.

Doc Impact:
new feature will require documentation in client and admin guides

Support Impact:
minimal support impact

Installation Impact:
None`

Admin/Config Impact:
No new admin functionally

I18N/L10N Impact:
None (only log strings)

Security Impact:
No security policys or class loader will be set up for the broker. This means that the client and the broker can affect each other. It is unlikely that this will cause a security issue since they will be operating as a single process.

Risks:
No obvious risks
