Synopsis:

UMS improvements for the crane release.

Description:

UMS will include the following improvements for the crane release. The designed usage of the services introduced in this document is through a browser interface for the crane release. They are designed to be used by people, not computer programs. The are intended for information look up, debugging, and trouble shooting purposes. The output format are classified as unstable and may be changed in the future releases.

1 Broker monitoring service.
This feature provides broker monitoring information through the UMS interface. An user may request broker information by entering the service URL into a browser's location bar. Upon received a monitoring service request, the monitoring service implementation would respond to a request in the HTTP response message if a request is accepted.

The output of the service is set in the HTTP response message body. The output is in xml format (classification: unstable, no schema binding) and would be displayed in the browser's window.

The service name is defined as “getBrokerInfo” service.

The service url is defined as follows (to be finalized in the functional spec).
http://<host>:<port>/<ums_context>/simple?service=getBrokerInfo&cmd=getDestinations

For example, the following is an example URL entered in the browser's location bar.
http://localhost:8080/ums/simple?service=getBrokerInfo&cmd=getDestinations

The detailed broker monitoring service information is at the link here:
http://jpgserv.red.iplanet.com/not/MQCrane/engineering/onepagers/ums/ums-monitoring.html


2. UMS utility services.

The following UMS utility services will be reviewed/updated and made public in the crane release.

2.1 ping service– ping broker through the UMS ping service.

http://<host>:<port>/<ums_context>/simple?service=ping

For example, the url (entered in a browser location bar) below would invoke the UMS (deployed on niagra2) ping service.
http://niagra2:8080/ums/simple?service=ping

The following output would be displayed in the browser window:
Broker is alive, round trip = 332 milli secs.

2.2 debug service – turn UMS service debug logging on/off with an url entered in a browser's location bar.

http://<host>:<port>/<ums_context>/simple?service=debug&debug=<true|false>

For example:
http://niagra2:8080/ums/simple?service=debug&debug=true

The following output would be displayed in the browser window:
service=debug, debug=true

2.3 ssend service– send a simple text message with an url entered in a browser's location bar.

http://<host>:<port>/<ums_context>/simple?service=ssend&destination=<string>&text=<string>[&domain=<queue|topic>]

For example:
http://niagra2:8080/ums/simple?service=ssend&destination=testq&text=”HelloWorld”

The above would send a text message with message body set to “hello world” to the queue destination named testq.
The following output would be displayed in the browser window:
Message sent: "hello world", destination = testq, domain=Queue

2.4 sreceive service– receive a simple text message with an url entered in a browser's location bar.

http://<host>:<port>/<ums_context>/simple?service=sreceive&destination=<string>[&domain=<queue|topic>]

For example:
http://niagra2:8080/ums/simple?service=sreceive&destination=testq

The above would receive a message (if available) from the queue destination named testq.
The following output would be displayed in the browser window if there is a “hello world” message in the queue:
"hello world"

2.5 getConfiguration service. - get the UMS configuration data with an url entered in a browser's location bar.

http://<host>:<port>/<ums_context>/simple?service=getConfiguration

For example:
http://niagra2:8080/ums/simple?service=getConfiguration
