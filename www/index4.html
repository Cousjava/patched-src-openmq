Overview  
 

Welcome to Open Message Queue

Open Message Queue (Open MQ) is a product that implements and facilitates message transfer between applications. The Open MQ project is the ongoing development of features, functions, ideas, and patterns that are useful for Open MQ, and sometimes similar products. This is a community of developers and users who have a shared interest in this product, or products and technologies like this. The Open Message Queue product is the community version of Sun Java System Message Queue. Complete downloads are available for Solaris Sparc, Solaris x86, Linux, and Windows, and we supply a Java only package for use on other platforms (such as Macintosh and AIX). The Open Message Queue Source code for the project is available as a single .zip archive file.

The product Open MQ provides a convenient way to connect your applications together so that they can run independently from each other. Open MQ can be used as a buffer between applications that can produce and consume messages and different rates. Open MQ can also distribute messages in a prescribed way. This is flexible and can be changed without impacting the applications that are sending or receiving messages. Open MQ implements a variety of message delivery patterns that allow it to be customized to meet a wide variety of requirements involving speed, buffer storage, resiliency, and stability and independence. Open Message Queue 4.3 -- Feature Freeze  
 

Open MQ 4.3 is now at Feature Freeze. We have the first iteration of all the features that are scheduled to be included in this release.

  • We are pleased to introduce UMS -- an extremely simple Web Client API for accessing JMS. There are two access methods for this new interface: You can use XML Documents or URL query strings to access the JMS destinations via HTTP or HTTPS. Read more about this, at the UMS feature description page.
  • We have introduced new install bundles which include an Open Installer UI. This provides a UI for the community installs. Other features include: Product Registration, unattended install and configuration, and text only install (Unix only). You can download these, from this link.
 Quick Introduction  
 

Open message queue is an enterprise quality, production ready, scalable messaging server. It provides a complete Java Message Service (JMS) implementation for message oriented system integration. In addition, Open MQ provides the additional enterprise features that are necessary for enterprise deployments, large and small. It gets its roots from Java Message Queue and provides all the features, functions and capabilities of the currently available licensed product: Java System Message Queue.

(Sun product support is not available for Open Message Queue, however we welcome your feedback and endeavor to provide assistance and incorporate your feedback as we continually improve the project. If you are interested in commercial versions of these products and features, please see the links section below)

Do you use Open Message Queue -- or its commercial sister, Java System Message Queue? Why not drop us a note about what you're doing with this product? We're not running any promotions, but we'd be grateful to hear about your experience with the product. Send e-mail to dev@mq.java.net

Open Message Queue is integrated into several community and commercial class products. It is the Java EE Java Message Service (JMS) provider for Project GlassFish, and it is the reference implementation JMS Provider for Java Platform, Enterprise Edition. If you are interested in including Open MQ in your product, we'd be happy to work with you. Please drop us a note on the user alias.

If you are interested in using a supported version of Open Message Queue, you should navigate over to the Sun Java System Message Queue product pages. Commercially licensed copies are available for download. Sun offers an array of plans that will provide you with flexible payment and support options, to meet your requirements. Please remember, features in the commercial product may vary from Open MQ.
